# Design README

这个目录现在的使命，不再是延续旧的 Portfolio / My Work 叙事，而是为新的 **PM Workspace** 提供可视化设计基线。

## 当前设计目标

SwissArmyPM 的设计应围绕一个问题展开：

**项目经理如何在最短时间内维护和理解一个项目的当前事实。**

## 设计中心

新的核心设计对象不是 Dashboard、Portfolio、My Work，而是：

- 项目列表
- 项目工作台
- 项目画布
- 干系人
- 时间规划表
- 风险登记册
- 工作包
- 资料/证据

## 推荐页面结构

### 1. 项目列表页

用于进入某个项目，不强调 PMO 风格组合治理。

### 2. 项目工作台首页

展示该项目最关键的当前信息：

- 当前阶段
- 关键交付时间
- top 风险
- 最近更新
- 关键干系人
- 最近证据

### 3. 标签式核心页

每个项目下至少包含：

- Canvas
- Stakeholders
- Timeline
- Risks
- Work Packages
- Evidence

### 4. 快速录入 / 快速编辑区

设计必须支持：

- 快速新增
- 快速改字段
- 快速查看上下文

不要把 PM 逼进复杂企业表单里。

## 设计原则

### 快速理解优先

项目经理打开页面后，应在几十秒内理解项目当前状态。

### 编辑成本低

录入和更新必须足够轻。

### 结构清晰

每类信息都要有清晰落点，避免继续依赖碎片文档或聊天记录。

### 证据可关联

后续涉及关键结论时，设计上必须预留来源展示位。
