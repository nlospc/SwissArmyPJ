# 项目概述

SwissArmyPM 正在被重新收敛为一个**只服务项目经理**的桌面工作台。

它的目标不是做通用协作系统，也不是继续扩展成组合治理平台，而是让项目经理在一个独立工作台里，快速维护和查看项目关键事实。

## 当前产品定义

一句话：

**一个面向项目经理的、证据驱动的项目事实工作台。**

项目经理应该能在这里快速完成这些事：

- 维护项目画布
- 维护干系人
- 维护时间规划表
- 维护风险登记册
- 维护工作包
- 保存和查看会议纪要、邮件、IM 片段等资料/证据
- 在未来基于证据回答关键问题并生成建议更新

## 当前核心模块

### 1. 项目列表 / 项目入口

用于进入具体项目，而不是做 PMO 式组合总览。

### 2. 项目工作台

每个项目都应该有统一工作台，至少包含：

- 项目画布
- 干系人
- 时间规划表
- 风险登记册
- 工作包
- 资料/证据

### 3. 快速录入与快速更新

产品第一优先级是让项目经理可以低成本维护项目信息。

### 4. 证据层

会议纪要、邮件、IM 关键对话、文档摘录等，先作为证据沉淀。

### 5. AI/Agent 插口

未来 Agent 的职责是：

- 从证据中抽取候选事实
- 发现变更与冲突
- 提出结构化更新建议
- 回答问题时给出来源

## 当前不再作为主线的方向

以下内容如果仍在仓库中出现，应理解为历史实现，不再代表产品中心：

- Portfolio Dashboard
- My Work / Pomodoro
- 个人待办中心
- 多角色 PMO 平台
- 通用项目协作平台

## 当前设计原则

### 项目经理唯一核心用户

产品体验只围绕项目经理设计。

### 桌面独立成立

即使没有 Jira、MS Project、邮件系统深度集成，产品也应先成立。

### 结构化优先

先把核心项目对象做扎实，再叠加 AI。

### 来源可追溯

未来关键问题必须支持“答案 + 来源”。

### PMBOK 设计原则

- 责任感地管理干系人
- 主动推进关键事项
- 聚焦项目价值
- 持续整合项目真相

### 快速更新优先于复杂流程

如果更新速度慢、录入负担重，项目经理不会持续使用。

### 证据增强而非喧宾夺主

借鉴 `gbrain` 的证据沉淀、引用回溯、去重导入和后台整理思路，
但产品中心仍然是项目经理的结构化项目工作台。
